## Why

To begin the development of the Medical Booking system, we need a solid foundation based on the decided tech stack. This includes a robust backend (NestJS) and two separate frontends (Patient Portal and Admin Dashboard) using Next.js. This setup ensures we can start implementing features localized to each user role independently.

## What Changes

- Initialize a NestJS project in the `server` directory.
- Initialize a Next.js project in the `apps/portal` directory (Patient Portal).
- Initialize a Next.js project in the `apps/admin` directory (Admin Dashboard).
- Configure Tailwind CSS and TypeScript for all frontend projects.
- Setup basic folder structure for TanStack Query and state management.

## Capabilities

### New Capabilities

- `backend-foundation`: Basic NestJS setup with structured modules.
- `portal-frontend-foundation`: Next.js setup with Tailwind and basic patient-facing pages.
- `admin-frontend-foundation`: Next.js setup with Tailwind and basic admin-facing pages.
